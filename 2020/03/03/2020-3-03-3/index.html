闲话

听过Node.js的一部分人会认为这是一种语言，常常看见有人问”NodeJS与Python之间哪种语言更好”这种问题。实际上，Node.js并不是一种语言，而仍然是JavaScript脚本，Node.js只不过是类似于一种运行环境或者说是一个平台，使用JS构造出来的，Node.js让JS可以在客户端与服务端之间来回跑。

简单来说Node.js就是可以运行在服务端的JavaScript，是一个事件驱动I/O服务端的js环境，基于谷歌的V8所带来的超高性能，速度极快。

为什么JavaScript是单线程

JavaScript其主要作用是与用户互动以及操作文档对象模型（DOM），如果是多线程，那在同时对DOM同一个节点增加删除的时候将会不好控制，所以出于复杂的同步问题考虑，JS从诞生就决定了其核心为单线程。

为了利用多核CPU的计算能力，HTML5提出了WebWorker标准，它允许JS脚本创建多个进程，但是子线程完全受控于主线程，而且不得操控DOM。所以即便是这个新标准也没有改变JS是单线程的本质。

异步核心-任务队列

等待一个任务完成再继续下一个任务这种做法使得程序在等待浪费时间，JS的设计者也同样意识到了这一点，所以JS的主线程可以不管IO设备，挂起等待中的任务去运行下一个任务，待到IO设备返回结果之后，再回头把挂起的任务继续执行下去。

具体来说，异步执行的运行机制如下：

  1. 所有同步任务都在主线程上执行，形成一个执行栈(execution context stack).
  2. 主线程之外，存在一个任务队列。只要异步任务有了运行结果，就在队列中放置一个事件。
  3. 一旦执行栈之中的所有同步任务执行完毕，系统就会读取任务队列中的事件，看看哪些对应的异步任务已经结束等待状态，便将其放入执行栈可是执行。
  4. 主线程会不断重复第三步。

在JS中任务被分为两种。

  • 同步任务(synchronous)：在主线程上排队执行的任务，只能等待前一个任务完成才能进行下一个任务。
  • 异步任务(asynchronous)：不进入主线程，进入任务队列(task queue)，只有通知主线程某个异步任务可以执行了，该任务才会进入主线程执行。

回调(callback)

setTimeout( () => { console.log(1); }, 0 );

console.log(2);

上面的代码中无论如何将定时器中的0改变，控制台输出顺序一定会是2 1，因为只有在执行完第二行之后系统才回去执行任务队列中的回调函数。因为回调时被挂起的任务，只能是在主线程最早可得的空闲时间来执行。

需要注意的是，setTimeout只是将事件插入了任务队列，必须等待当前代码执行栈执行完毕，主线程才会去执行它指定的回调函数。如果当前代码的耗时很长，有可能需要等待很久，所以没有办法保证，回到函数一定会在定时器指定的时间执行。
